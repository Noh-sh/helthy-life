# ИИ-Навык (AI Skill): Next.js App Router & React Best Practices

Этот документ содержит стандарты индустрии (Industry Best Practices) для работы с Next.js (App Router) и React. ИИ обязан строго следовать этим правилам при написании логики и архитектуры.

## 1. Базовые принципы (Core Principles)
*   **Серверные компоненты по умолчанию (RSC):** Все компоненты должны быть серверными (Server Components). Директиву `'use client'` использовать ТОЛЬКО там, где необходима интерактивность (хуки `useState`, `useEffect`, обработчики событий `onClick`).
*   **Функциональный код:** Писать только функциональные компоненты (Functional Components). Никаких классовых компонентов.
*   **Чистые функции:** Функции должны быть максимально чистыми и предсказуемыми.

## 2. Архитектура Next.js (App Router)
*   **Роутинг:** Вся маршрутизация находится исключительно в папке `app/`. Папку `pages/` не использовать.
*   **Получение данных (Data Fetching):** Использовать `async/await` прямо в серверных компонентах. Для мутаций данных использовать Server Actions (с директивой `'use server'`).
*   **Оптимизация:** Всегда использовать встроенные компоненты: `<Image />` (`next/image`) для картинок и `<Link />` (`next/link`) для навигации.
*   **SEO:** Использовать Metadata API (экспорт объекта `metadata`) в каждом `page.tsx` для правильного SEO.

## 3. Структура файлов и папок
*   **Колокация (Colocation):** Файлы `loading.tsx` (скелетоны загрузки), `error.tsx` и `not-found.tsx` должны лежать рядом с соответствующими `page.tsx`.
*   **Shared Components:** Переиспользуемые UI-компоненты (кнопки, карточки) хранить в `src/components/`.
*   **Локальные компоненты:** Компоненты, относящиеся только к одной странице, хранить в папке `_components` внутри директории этой страницы в `app/`.

## 4. Состояние и Управление (State Management)
*   **Глобальное состояние:** Держать минимальным. Избегать "prop drilling". Использовать Zustand или React Context только при крайней необходимости.
*   **URL как State:** Для фильтров каталога и поиска (`smart_search.md`) использовать параметры URL (Search Parameters) через `useSearchParams`, а не локальный state, чтобы ссылками можно было делиться.

## 5. Качество кода (Clean Code)
*   **Именование:** `PascalCase` для компонентов и файлов компонентов. `camelCase` для функций и хуков.
*   **Типизация:** Использовать TypeScript. Предпочитать `interface` вместо `type`. Избегать `any`.
*   **Экспорты:** Использовать именованные экспорты (`export const MyComponent`), избегать `export default` (кроме страниц Next.js, где это обязательно).
