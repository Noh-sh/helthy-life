# ИИ-Навык (AI Skill): Mobile-First Разработка

Этот документ устанавливает строгие правила адаптивной верстки. ИИ обязан применять подход **Mobile-First** при написании любого CSS/компонента.

## 1. Базовый принцип (Mobile-First)
*   **Сначала смартфоны:** CSS код пишется изначально для мобильных экранов (базовые стили без медиа-запросов).
*   **Desktop-later:** Стили для планшетов и ПК добавляются только через медиа-запросы `min-width` (например, `@media (min-width: 768px)`).
*   **НИКАКИХ `max-width`** для базовой верстки, кроме исключительных случаев.

## 2. Размеры экранов (Breakpoints)
Использовать стандартизированные брейкпоинты:
*   **Мобильные (Base):** от 320px до 767px (дефолтные стили).
*   **Планшеты (Tablet):** `@media (min-width: 768px)`.
*   **Десктоп (Desktop):** `@media (min-width: 1024px)`.
*   **Широкие экраны (Large):** `@media (min-width: 1280px)`.

## 3. Мобильные компоненты (App-like UX Patterns)
В 2024+ году мобильные сайты должны ощущаться как нативные приложения (iOS/Android).
*   **Отказ от классического Бургер-меню:** Для основных разделов использовать **Bottom Navigation Bar** (Нижняя панель с иконками: Главная, Каталог, Корзина, Профиль), как в Instagram или Telegram. Бургер-меню (Hamburger) допускается ТОЛЬКО для второстепенных ссылок (О нас, Контакты, Документы).
*   **Bottom Sheets (Шторки) вместо Модальных окон (Modals):** На мобильных экранах всплывающие окна по центру экрана неудобны. Все интерфейсы (выбор фильтров, корзина, подтверждения) должны выезжать снизу в виде "шторки" (Bottom Drawer), чтобы их было легко закрыть свайпом вниз или дотянуться большим пальцем.
*   **Липкие элементы (Sticky Elements):** Главный призыв к действию (Кнопка "Купить" на карточке товара) должен быть зафиксирован (`position: fixed; bottom: 0;`) над нижней кромкой экрана при прокрутке.
*   **Сенсорные экраны (Touch Targets):** Минимальный размер кликабельных элементов должен быть `44px` на `44px` во избежание промахов ("Fat finger" rule).
*   **Жесты (Gestures):** Поддержка горизонтального свайпа (Swipe) для галереи товаров навигации назад.

## 4. Оптимизация и Производительность
*   **Изображения:** Обязательно использовать атрибут `loading="lazy"` и предоставлять оптимизированные форматы (WebP) через компонент `next/image`.
*   **Hover-эффекты:** На мобильных устройствах (где нет курсора) hover-эффекты (например, 3D-карточки) не должны мешать чтению. Отключать или заменять их на эффекты по клику/тапу, используя медиа-запрос `@media (hover: hover)`.
