# ИИ-Навык (AI Skill): Доступность (Accessibility / a11y)

Этот документ описывает строгие правила доступности (Web Accessibility) для интерфейсов. Поскольку проект связан со здоровьем, им будут пользоваться люди с ограничениями по зрению или моторике. ИИ обязан соблюдать стандарты WCAG.

## 1. Контрастность и Цвет
*   **Контраст текста:** Текст должен иметь высокий контраст с фоном (минимум 4.5:1 для обычного текста). Серый текст на сером фоне СТРОГО ЗАПРЕЩЕН.
*   **Цвет — не единственный индикатор:** Если показывается ошибка (Error), нельзя полагаться только на красный цвет. Обязательно должна быть поясняющая иконка (например, `⚠`) и текстовое описание.

## 2. Структура и Семантика (HTML5)
*   **Семантические теги:** Использовать правильные теги вместо бесконечных `<div>`. 
    *   `<header>`, `<nav>`, `<main>`, `<section>`, `<article>`, `<footer>`.
*   **Кнопки vs Ссылки:** 
    *   Если действие меняет состояние (открыть модальное окно, добавить в корзину) — использовать `<button>`.
    *   Если действие переводит на другую страницу — использовать `<a>` (или `<Link>` в Next.js).
*   **Заголовки:** Иерархия должна быть строгой: `H1` -> `H2` -> `H3` без пропусков. Только один `H1` на страницу.

## 3. Поддержка Скринридеров (Screen Readers)
*   **Изображения:** У каждого `<Image>` (или `<img>`) обязательно должен быть осмысленный атрибут `alt=""`. Если изображение несет чисто декоративный характер (узор, абстракция), `alt` должен быть пустым `alt=""`, чтобы скринридер его проигнорировал.
*   **ARIA-атрибуты:** Для сложных кастомных компонентов (капсульный навигатор, аккордеоны, модальные окна) обязательно использовать `aria-expanded`, `aria-hidden`, `aria-label`.

## 4. Навигация с клавиатуры
*   **Focus States:** Все интерактивные элементы (кнопки, ссылки, поля ввода) должны иметь четко видимый `focus` стейт при навигации через клавишу `Tab`.
    *   Tailwind классы: `focus:outline-none focus:ring-2 focus:ring-emerald-500`.
*   **Пропуск к контенту:** Рекомендуется внедрять невидимую кнопку "Skip to content" в самом верху страницы для клавиатурных пользователей.
