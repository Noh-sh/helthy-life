# ИИ-Навык (AI Skill): Интерактивные Компоненты (Design Components)

Этот документ описывает логику создания сложных, фирменных UI-компонентов проекта.

## 1. Капсульная Навигация (Capsule Navigator)
Обязательный элемент главной навигации.
*   **Формат:** Меню в виде "Капсулы" со скругленными краями (`border-radius: 9999px` / `rounded-full`). Фон навигатора должен слегка выделяться (например, светло-серый `#F1F5F9`).
*   **Индикатор (Sliding Button):** Выбранный пункт меню должен иметь свой собственный фон (например, белый `bg-white` с легкой тенью `shadow-sm`), который визуально выглядит как кнопка внутри капсулы.
*   **Магическая Анимация (Magic Motion):** При клике на другой пункт меню, фоновый индикатор (активный стейт) не должен просто появляться, он должен **плавно переезжать (скользить)** от старой иконки к новой. Для реализации этого эффекта ИИ ОБЯЗАН использовать библиотеку `framer-motion` (свойство `layoutId`). Этот же эффект должен применяться в мобильном Bottom Navigation Bar.
*   **Типографика:** Обязательно использование заглавных букв (`uppercase`) и шрифта `Semi Bold`.

## 2. Интерактивные Карточки (3D Services Cards)
Применяются на странице `/services` и для выделения ключевых продуктов.
*   **Hover-эффект (3D):** При наведении курсора карточка должна плавно подниматься вверх (CSS: `transform: translateY(-8px)` или Tailwind `hover:-translate-y-2`).
*   **Плавность (Transition):** Любая анимация карточки должна иметь `transition-all duration-300 ease-in-out`.
*   **Тень (Shadow):** В состоянии покоя карточка имеет легкую тень (`shadow-sm`). При наведении тень должна становиться объемной и мягкой (`hover:shadow-xl` или `hover:shadow-lg`), чтобы подчеркнуть эффект "отрыва" карточки от фона.

## 3. Общие правила компонентов
*   **Скругления (Border Radius):** Для всех карточек, модальных окон и кнопок (кроме капсул) использовать консистентное скругление: `16px` (`rounded-2xl`). Острые углы запрещены!
*   **Кнопки (Buttons):** Главные кнопки (Call to Action) должны быть залиты Primary цветом (Изумрудным) и иметь легкое изменение яркости при hover-эффекте (`hover:brightness-110` или аналогичный класс изменения оттенка).
