# Техническая Спецификация Проекта

Данный документ определяет техническую архитектуру, структуру данных и логику взаимодействия компонентов системы.

## 1. Технологический Стек
*   **Frontend:** Next.js 14+ (App Router) — для обеспечения высокой производительности и SEO.
*   **Стилизация:** Vanilla CSS с использованием CSS Variables для тем оформления.
*   **Backend:** Next.js API Routes (Serverless Functions).
*   **База данных:** PostgreSQL (рекомендуется использование через Supabase для масштабируемости).
*   **Аутентификация:** NextAuth.js или Supabase Auth.
*   **Платежи:** Интеграция с ЮKassa / Stripe API.

## 2. База Данных (Схема и Связи)
> [!IMPORTANT]
> **MVP Этап (Первая версия):** На начальном этапе разработки (Frontend-first) реальная база данных PostgreSQL **НЕ используется**. Все данные товаров и курсов берутся из локальных Mock-файлов (`data_models.md`), а пользовательские данные (Корзина, Избранное, статус авторизации) сохраняются в **LocalStorage** браузера.

Когда проект перейдет в стадию Backend-разработки, система будет строиться на реляционной модели данных:

*   **Таблица `Products` (Товары):** `id`, `name`, `description`, `price`, `stock_quantity`, `category_id`, `substances[]` (массив активных веществ).
*   **Таблица `Courses` (Курсы):** `id`, `title`, `description`, `price`, `author_info`, `content_json` (структура уроков).
*   **Таблица `Users` (Пользователи):** `id`, `email`, `password_hash`, `full_name`, `created_at`.
*   **Таблица `Orders` (Заказы):** `id`, `user_id`, `total_amount`, `status` (pending, paid, failed), `items_json` (список купленного).
*   **Связи:**
    *   `Orders.user_id` -> `Users.id` (Один пользователь — много заказов).
    *   `Products.category_id` -> `Categories.id`.

## 3. Страницы и Навигация
Используется файловый роутинг Next.js:

*   `/` — Главная страница (Поиск и популярные товары/курсы).
*   `/about` — Страница "Обо мне" (информация об эксперте/компании).
*   `/services` — Страница "Сервисы" (список услуг и 3D-карточки).
*   `/contacts` — Страница "Контакты" (информация для связи и формы).
*   `/catalog` — Каталог БАДов (Фильтрация на стороне клиента + SSR).
*   `/catalog/[id]` — Динамическая страница товара.
*   `/courses` — Каталог программ обучения.
*   `/courses/[id]` — Страница конкретного курса.
*   `/cart` — Корзина (хранение в LocalStorage/SessionStorage до авторизации).
*   `/checkout` — Защищенная страница оформления заказа.
*   `/profile` — Личный кабинет (доступ к курсам и истории).

## 4. Backend / API Эндпоинты
Взаимодействие фронтенда с данными:

*   `GET /api/products` — получение списка товаров с фильтрацией.
*   `GET /api/products/[id]` — детальные данные о товаре.
*   `POST /api/orders` — создание нового заказа и инициализация платежа.
*   `POST /api/payment/webhook` — обработка подтверждений от платежной системы.
*   `POST /api/contact` — отправка данных из формы обратной связи в Telegram/Email администратора.

## 5. Связи между частями проекта
Взаимодействие происходит по следующей схеме:

1.  **Клиент (Frontend):** Собирает данные (поиск, корзина) и отправляет запросы к API.
2.  **API Layer (Next.js Routes):**
    *   Проверяет права доступа (Auth).
    *   Запрашивает данные из базы (PostgreSQL).
    *   Обращается к внешним API (Платежи, Мессенджеры).
3.  **База Данных:** Хранит контент и состояние заказов.
4.  **Внешние сервисы:** 
    *   **Платежный шлюз 1 (Локальный):** ЮKassa или аналог для приема рублевых карт.
    *   **Платежный шлюз 2 (Международный):** Stripe или PayPal для зарубежных покупателей курсов (защита от отказов по иностранным картам).
    *   Система подтверждает оплату -> API обновляет статус в БД -> Клиент получает доступ к курсу.
## 6. Инфраструктура и Инструменты Разработки
*   **Контроль версий:** Git.
*   **Репозиторий:** GitHub (хранение исходного кода, командная работа).
*   **Хостинг / Деплой (Frontend):** Vercel (обеспечивает автоматическую сборку и развертывание при пуше в основную ветку GitHub, идеальная интеграция с Next.js).
*   **Пакетный менеджер:** npm (или pnpm для более быстрой установки зависимостей).
*   **Линтинг и Форматирование:** ESLint и Prettier (для поддержания единого стиля кода).
