# Спецификация функции: Регистрация и Личный Кабинет (User Auth & Profile)

## 1. Поведение функции
Эта система управляет доступом пользователей к купленным курсам и истории заказов. Для обеспечения максимального удобства и безопасности используется **Passwordless Auth** (авторизация без паролей).

*   **Авторизация (Login/Signup):** Пользователь вводит свой Email. На почту приходит одноразовый 4-значный код (OTP) или Magic Link (магическая ссылка). Пользователь вводит код и сразу попадает в кабинет.
*   **Доступ к курсам:** В кабинете отображаются все купленные курсы в виде сетки карточек.
*   **Таймер доступа:** У каждого курса есть визуальный индикатор (прогресс-бар или текст), показывающий, сколько дней доступа осталось (например, "Осталось 14 дней").
*   **История покупок:** Раздел со списком всех заказанных БАДов и их статусами (Собирается, В пути, Доставлен).
*   **Отслеживание (Tracking):** Для отправленных заказов обязательно выводится **Трек-номер**, кнопка "Скопировать" и прямая ссылка на сайт транспортной компании (СДЭК/Почта России).

## 2. Данные пользователя
*   **Учетная запись:** Email (используется как ID), Имя, Телефон (опционально для доставки).
*   **Модуль "Мои курсы":** Массив купленных курсов с датой окончания доступа.
*   **Модуль "Мои заказы":** Массив заказов физических товаров со статусами отслеживания и трек-номерами транспортных компаний.

## 3. Исключительные случаи и Алгоритмы восстановления

### 3.1. Код авторизации не пришел (Письмо в СПАМе)
*   **Ситуация:** Пользователь ввел Email, но код не приходит более 1 минуты.
*   **Инструкция по исправлению:**
    *   Показать таймер (60 секунд) до возможности запросить код повторно.
    *   Вывести яркую подсказку: "Письмо может попасть в папку СПАМ или Промоакции. Пожалуйста, проверьте их".
    *   Если после 2-х попыток код не найден, предложить альтернативу: "Связаться с поддержкой через Telegram для ручного входа".

### 3.2. Доступ к курсу закончился
*   **Ситуация:** Пользователь заходит в кабинет, а срок действия курса (например, 6 месяцев) истек.
*   **Инструкция по исправлению:**
    *   Курс не исчезает из кабинета! Он переходит в состояние "Завершен / Доступ закрыт" (серая обложка).
    *   При клике на курс открывается окно (Modal) с предложением: "Срок доступа истек. Хотите продлить доступ со скидкой 50% для выпускников?"
    *   Кнопка "Продлить доступ" ведет сразу на защищенную страницу оплаты.

### 3.3. Вход с другого устройства (Security)
*   **Ситуация:** Пользователь пытается зайти с нового браузера или телефона.
*   **Инструкция по исправлению:**
    *   Поскольку используется OTP (одноразовый код на почту), каждый новый вход требует ввода кода. Это автоматически защищает аккаунты от передачи третьим лицам.
